Segurança
A api da asKate foi desenvolvida pensando na fácil utilização e na escalabilidade, mas principalmente na segurança. Nenhuma informação crítica das plataformas trafega pelas requisições e respostas da api: Tudo é feito por meio de tokens internos da asKate, passado pelo cabeçalho da requisição como argumento "x-token". Existem 3 tokens utilizados dentro na plataforma:
Token:
Regenerado em todo login. É utilizado por padrão pela api em todo seu funcionamento

Api Secret Token:
Token fixo que é trocado pelo usuário quando bem entender, por meio da rota "/user/reset_api_token"

Api Access Token:
Token fixo que é trocado pelo usuário quando bem entender, por meio da rota "/user/reset_api_token"

(Tanto api_secret_token quanto api_access_token foram criados para uso em api's futuras)
(Todos os tokens são constituídos de 60 caracteres aleatórios. Ex.: f6248e96eb7374610693edc36962825466a8bcd5e72c7be11b1507d220f4, apresentando 3660 possibilidades)
3660 = 2,388,636,399,360,109,977,557,402,041,718,133,080,829,429,159,844,757,507,642,063,199,359,529,632,522,467,783,435,119,230,976


Sincronização
Para garantir a integridade do sistema interno, no momento de sincronização como uma plataforma externa, é gerado um sync_token que é utilizado exclusivamente para a sincronização. Depois deste momento, este token não terá mais utilidade e será excluído do banco de dados.
